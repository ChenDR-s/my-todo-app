# 待办事项管理器多平台版 - 设计文档

## 项目概述
将现有命令行待办事项管理器升级为多平台应用：
1. **简化PySide6桌面应用**（EXE可执行文件）
2. **轻量级网页版**（后续可部署到服务器）
3. **主从API架构**：桌面应用为主，网页版为从

## 核心需求
- ✅ 3列卡片式界面（待办、进行中、已完成）
- ✅ 显示任务内容、日期、标签
- ✅ 双击卡片编辑，拖放移动
- ✅ 浅色主题，统一图标
- ✅ 数据兼容现有tasks.json格式
- ✅ 用户数<10，稳定快速低成本

## 技术架构

### 系统架构
```
桌面应用 (PySide6) ←HTTP API→ 网页版 (Flask)
        ↓                           ↓
    tasks.json                  浏览器访问
```

### 项目结构
```
todo_manager/
├── core/           # 共享核心
├── desktop/        # 桌面应用
├── web/           # 网页版
├── resources/      # 共享资源
├── tasks.json     # 数据文件
└── run.py         # 启动脚本
```

## 数据模型

### 任务格式（扩展现有）
```json
{
  "id": 1,
  "content": "任务内容",
  "status": "todo",  // "todo", "in_progress", "done"
  "created_at": "2024-03-11T10:30:00",
  "tags": ["标签1", "标签2"]
}
```

### 兼容性处理
- 旧任务自动添加status="todo"
- 保持向后兼容

## 桌面应用设计

### 界面布局
```
[菜单栏] [工具栏]
[待办事项列] [进行中列] [已完成列]
[状态栏：网页地址]
```

### 核心功能
1. 3列卡片显示（QListWidget或自定义组件）
2. 添加任务对话框
3. 删除任务确认
4. 任务移动（拖放或按钮）
5. 内置Web服务器（localhost:5000）
6. 打开网页版按钮

## 网页版设计

### 技术栈
- 后端：Flask（极简）
- 前端：HTML + CSS + JavaScript（原生）
- 通信：REST API

### 功能范围
1. 只读查看任务
2. 3列卡片显示
3. 自动刷新（每10秒）
4. 连接状态提示

## API设计

### 端点
- `GET /api/tasks` - 获取所有任务
- `POST /api/tasks` - 添加任务（转发到桌面应用）
- 网页版不直接修改数据，通过桌面应用操作

## 工作流程
1. 用户运行TodoManager.exe
2. 启动桌面界面和Web服务器
3. 桌面应用管理任务
4. 浏览器访问http://localhost:5000查看任务
5. 网页版自动同步数据

## 注意事项
1. 网页版为只读，修改需通过桌面应用
2. 确保端口5000可用
3. 数据文件位于应用同级目录
4. 首次运行自动迁移旧数据
